Бот для отображения актуального расписания занятий университета с интеграцией Google Sheets, системой уведомлений и ролевой моделью.

## 📋 Возможности

### Для студентов:
- `/start` — приветствие и выбор группы
- `/schedule` — расписание на сегодня
- `/schedule завтра` — расписание на завтра
- `/schedule неделя` — расписание на всю неделю
- `/next` — информация о следующей паре
- `/search <код>` — поиск группы по коду
- `/notify on|off|time` — управление уведомлениями
- `/changes subscribe` — подписка на изменения в расписании

### Для руководителей:
- `/admin sync` — принудительная синхронизация с Google Sheets
- `/admin edit <группа> <день>` — редактирование расписания
- `/admin broadcast <текст>` — рассылка всем пользователям
- `/admin logs` — журнал изменений

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
cd schedule-bot
go mod tidy
```

### 2. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и заполните значения:

```bash
cp .env.example .env
```

Необходимые переменные:
- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather
- `GOOGLE_API_KEY` — API ключ для доступа к Google Sheets
- `GOOGLE_SHEET_ID` — ID таблицы с расписанием
- `ADMIN_USER_IDS` — Telegram ID администраторов

### 3. Запуск

```bash
go run cmd/bot/main.go
```

### 4. Сборка для продакшена

```bash
go build -o schedule-bot cmd/bot/main.go
./schedule-bot
```

## 🔧 Технологии

- **Язык**: Go 1.21+
- **Telegram Bot API**: github.com/go-telegram-bot-api/telegram-bot-api/v5
- **Google Sheets**: google.golang.org/api/sheets/v4
- **База данных**: SQLite (dev) / PostgreSQL (prod)
- **Кэширование**: Встроенное с TTL
- **Логирование**: slog (Go 1.21+)

## 📝 Формат таблицы Google Sheets

Таблица должна иметь следующую структуру:
- Строка 1: Коды групп (8251160, 1251160...)
- Строка 2: Специальности (23.05.01, 44.03.04...)
- Столбец A: Дни недели
- Столбец B: Тип недели (в/н)
- Столбец C: Время начала
- Остальные столбцы: Данные занятий в формате "Дисциплина | Тип | Здание | Аудитория | Кафедра | Преподаватель | Примечания"

## 🔐 Безопасность

- Все чувствительные данные хранятся в переменных окружения
- Административные команды защищены проверкой Telegram ID
- Rate limiting для Google Sheets API
- Graceful shutdown при завершении работы

## 📊 Мониторинг

Логи выводятся в stdout в формате JSON. Уровень логирования настраивается через `LOG_LEVEL`:
- `debug` — подробная отладочная информация
- `info` — стандартные события (по умолчанию)
- `warn` — предупреждения
- `error` — только ошибки
